# Impeccable — Design Context

## Design Context

### Users
A single operator (a senior developer) plus AI agents that read/write project
state. Opened throughout the day to answer "where does everything stand?" across ~19 projects
and their integration coverage (Sentry, Zoho email). Primary job: **scan status at a glance**.
Secondary: expand a project to edit tasks / details.

### Brand Personality
Operational · precise · warm-blooded. It is a *command center*, not an enterprise BI tool —
it should feel like a personal mission-control console: dense, instrument-like, fast to read,
with a little warmth and character so it feels owned rather than generic.

### Aesthetic Direction
**Modern-minimal light, data-first — "Quiet" (Linear/Stripe school).** (Redesigned 2026-05-21
from the prior operator-dark skin; chosen from a 6-direction Hallmark mockup exploration in
`mockups/`. Earlier skins preserved in git history.) Cool gray-white canvas, soft 1px borders,
gentle radii (8–12px), low-spread shadows. Calm and premium: rows are separated cards that lift
subtly on hover; the active row gets an indigo edge. Inter Tight carries UI + headings (tight
tracking, semibold) and IBM Plex Mono carries labels, counts, and tabular data. Status colors
stay instrument-restrained (~10%), as soft tinted chips with a deeper text tone for AA, always
paired with a glyph/label. ONE responsive layout (no separate desktop/mobile DOM); the board
scans top-to-bottom and aligns into a real table via container queries when wide.

Anti-references (what it must NOT look like): cyan-on-black "AI dashboard"; glassmorphism/glow
decoration; pure white paper or pure black ink (everything tinted toward the cool 265° anchor);
loud gradients. The single accent is indigo (~270°), used sparingly (< 5% of any viewport).

### Design Principles
1. **Color means status.** Neutrals carry 90% of the surface; instrument colors (green/amber/
   blue/violet) appear only as status signal (~10%). Amber is the single interactive accent.
2. **Status is never color-only.** Every state pairs color with a glyph or label (✓ / · / text)
   — color-blind safe and the "color is not the only indicator" rule.
3. **One layout, mobile-first.** Base styles target small screens; container queries enhance to
   aligned table-columns when there's room. No `display:none` layout swaps.
4. **Density with air.** Tight rows separated by hairlines, not gaps; generous space only at
   structural seams (group headers, stat strip).
5. **Touch-grade everywhere.** 44px minimum hit targets; actions are always visible, never
   hover-only (touch has no hover).

### Type & Color System (implemented)
- Type (two roles): **Inter Tight** (UI + headings, semibold, tight tracking) + **IBM Plex Mono**
  (data, labels, counts, tabular grids). Mono is functional here (tabular alignment), not decorative.
- Color: OKLCH, cool-tinted gray-white paper/ink (hue ~250–265, very low chroma) + a single indigo
  accent (~270°), soft status chips (green/amber/blue/violet) with deeper text tones for AA, and
  three shadow tiers (sm/md/lg) for the lift-on-hover depth. Tokens in `app/globals.css`.
