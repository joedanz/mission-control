# What is Mission Control?

One place to see every project you're working on — and to let AI do some of the work.

Imagine you run a small workshop with **19 different projects** on the go at once. Some are
for clients, some are your own ideas, some are open-source. Each has a to-do list, a few
"plumbing" tasks (like hooking up payments or error-tracking), and a status — planning,
building, launched.

Holding all of that in your head, or in a dozen scattered notes, is impossible. **Mission
Control is the one screen that shows it all.** But it does something unusual on top of that:
it also lets **AI coding assistants pick up tasks and work on them by themselves**, while you
watch their progress, their cost, and their results in real time.

<Note>
**A mission-control dashboard, plus an autopilot.** Mission Control shows the status of every
project, and lets AI workers claim tasks, do them, and report back — all recorded in one
shared database that humans and machines read from together.
</Note>

## Two kinds of "user" share one system

  <Card title="You, a human" icon="user">
    **User type A** — You open the website in a browser, sign in with Google, and click around:
    review projects, add tasks, check what the AI is doing, and see what it cost.

    *→ via the web dashboard*
  </Card>
  <Card title="AI agents" icon="bot">
    **User type B** — AI assistants don't use a mouse — they use a typed command tool called
    `mc`. They read the same project list, claim tasks, and record their progress automatically.

    *→ via the mc command-line tool*
  </Card>
</CardGroup>

The key idea: **both humans and AI write to the exact same database.** Whatever the AI does
appears instantly on your screen, and whatever you change is instantly visible to the AI. One
source of truth, and everyone is looking at it.

On top of the project and task register, the dashboard also surfaces live signals for each
project: error tracking (via Sentry), email DNS health, and revenue (via Stripe) appear as extra
tabs once you set the matching project field. Each project also has an [MCP](/mcp) tab for
connecting it to outside tools — Linear, Slack, or any remote MCP server — that AI agents can then
call. See [The dashboard](/dashboard) for details.

And when a single task isn't enough, **[workflows](/workflows)** let you wire AI agents, tool calls,
branches, and human approval gates into a defined flow you draw on a canvas — orchestrating the agents,
not just watching them.
