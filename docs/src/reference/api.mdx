# The HTTP surface & its secrets

A handful of endpoints — split cleanly into a write-only telemetry door and read-only
session-guarded views.

| Endpoint | Auth | What it does |
|---|---|---|
| `POST /api/ingest` | Bearer `INGEST_TOKEN` | The telemetry door. Accepts `run.start`, `run.heartbeat`, `run.end`, and `event` bodies. Writes go *through* `lib/mutations` so the single-writer invariant holds. The **only** way activity gets in. |
| `GET /api/activity` | session-auth | The Mission tab's poll — events at or above `info` plus recent runs. Requires an allowed signed-in user, not the bearer. |
| `GET /api/board` | session-auth | The Kanban board's poll — custom tasks grouped per project (lean projection) plus active runs. `?project=<slug>` scopes it to one project (and returns all its Done tasks); no param returns every active project with each Done column capped. |
| `GET /api/runs/[id]` | session-auth | One run with its event trail, for the run drill-in page's poll. |
| `GET /api/cron/reap` | Bearer `CRON_SECRET` | Runs one reaper tick over HTTP — optional; the default is local `npm run reap`. |
| `/api/auth/[...all]` | — | BetterAuth — Google OAuth callback and session management. |
| `GET /api/projects/[slug]/composio` | session-auth | Returns `{ servers }` — the project's MCP connections (both `composio` and `remote` sources) and their statuses. `POST` with `{ action }` drives the lifecycle: `'connect'\|'status'\|'disconnect'` (need `toolkit`), `'add-remote'` (`{name,url,headers}`), `'remove-remote'` (`{name}`). |
| `GET /api/projects/[slug]/composio/catalog` | session-auth | Browse Composio's full live catalog. `?search` (fuzzy) and `?limit` page it; each toolkit carries a `featured` flag (MC's curated set) and a `connected` flag (the project already has an active connection). |
| `GET /api/projects/[slug]/workflows` | session-auth | The [Workflows](/workflows) tab's poll. Bare: the project's workflows with a latest-run summary. `?workflow=<slug>`: one workflow's graph plus the latest run's per-node step overlay. `POST` with `{ action }` drives the canvas: `'run'` enqueues a run (409 on the single-flight guard), `'approve'` decides a paused gate (`{runId,nodeId,decision,reason}`), `'save'` validates and persists an edited graph (422 on an invalid graph). The web tier never spawns — it only writes the `queued` row the workflow daemon claims. |
| `POST /api/workflows/[slug]/webhook` | HMAC `X-Hub-Signature-256` | The **event-trigger** door — public, **not** session-gated (the signature is the auth). A valid HMAC over the raw body, keyed by `WORKFLOW_WEBHOOK_SECRET`, enqueues a `trigger='event'` run carrying the body as `{{trigger.output.*}}`. Deliberate non-fires (inactive / not an event trigger / event-type filtered / single-flight) return `200 {fired:false}`; bad signature → `401`, bad JSON → `400`, unknown slug → `404`, unset secret → `503`. |
| `GET /api/projects/[slug]/errors` | session-auth | Live Sentry unresolved issues for the project. Requires `SENTRY_AUTH_TOKEN`, `SENTRY_ORG`, and `sentry_project_slug` on the project row. |
| `GET /api/projects/[slug]/email` | session-auth | Live DNS check (MX / SPF / DMARC) plus provider detection for the project's email address. |
| `GET /api/projects/[slug]/revenue` | session-auth | Live Stripe active subscriptions and computed MRR for the project. Requires `STRIPE_SECRET_KEY` and `stripe_site` on the project row. |

A telemetry post is a small typed JSON body with a `type` discriminator and a bearer header:

```bash
curl -X POST "$MC_INGEST_URL" \
  -H "authorization: Bearer $INGEST_TOKEN" \
  -H "content-type: application/json" \
  -d '{"type":"run.start","id":"<uuid>","agentLabel":"claude-code","workDir":"/path/to/repo"}'
# then, repeatedly:  {"type":"run.heartbeat","id":"<uuid>"}
# at the end:        {"type":"run.end","id":"<uuid>","status":"completed","tokensIn":4200,"costMicros":9100}
```

On `run.start`, if no `projectId` is given the server auto-associates the run with a project by
matching the hook's `workDir` against each project's `repo_path`.

## Environment variables

| Variable | Purpose |
|---|---|
| `DATABASE_URL` | Owner role locally (migrations + seed); the **app** role in Vercel. |
| `AGENT_DATABASE_URL` | The scoped agent role for `mc`. **Never** set in Vercel. |
| `INGEST_TOKEN` | Bearer secret the hooks send to `/api/ingest`. Write-only; set in Vercel too. |
| `CRON_SECRET` | Bearer the reaper cron sends, so the endpoint isn't publicly triggerable. |
| `BETTER_AUTH_SECRET` / `_URL` | Session signing secret and the canonical origin (app throws if the secret is missing). |
| `GOOGLE_CLIENT_ID` / `_SECRET` | Google OAuth credentials for the sign-in flow. |
| `ALLOWED_EMAIL` | The one address allowed to sign in. **Fail-closed** — empty rejects everyone. |
| `MC_INGEST_URL` / `MC_AGENT` | Where the hooks post, and the agent label they post under. |
| `MC_CLAUDE_BIN` / `MC_BIN` | Daemon-side: absolute paths that pin the `claude` executor and the `mc` CLI, so `npm run` can't shadow them with a stale `node_modules/.bin` copy. Set in the launchd plists. |
| `SENTRY_AUTH_TOKEN` | Sentry personal/integration token with `project:read` scope. Required for the Errors tab. |
| `SENTRY_ORG` | Sentry organization slug. Required alongside `SENTRY_AUTH_TOKEN`. |
| `SENTRY_BASE_URL` | Sentry API base URL. Optional — defaults to `https://sentry.io`. Override for self-hosted instances. |
| `STRIPE_SECRET_KEY` | Stripe API key. Read-only key recommended. Required for the Revenue tab. |
| `COMPOSIO_API_KEY` | Composio developer key. Resolved from the host environment at daemon spawn — never stored in the database. Required for the `mc mcp` Composio subcommands and the `/api/projects/[slug]/composio` routes. (The `remote` MCP source needs no broker key.) |
| `WORKFLOW_WEBHOOK_SECRET` | A single global secret that signs event-trigger webhooks (`X-Hub-Signature-256` = `sha256=HMAC-SHA256(secret, rawBody)`, GitHub-compatible). Never stored in the database; if unset, the webhook route returns `503`. |
| `MC_PUBLIC_BASE_URL` | Optional. The public origin `mc workflow webhook-url` prefixes onto the webhook path so it prints a full URL. |
